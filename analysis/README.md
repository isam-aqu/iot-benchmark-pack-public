# Analysis Artifact Layout

Generated cross-run outputs live under `analysis/`. Human-written procedures and interpretation live under `docs/`, while per-run raw data and provenance stay under `experiments/runs/<RUN_ID>/`.

## Directory Roles

- `analysis/w1/figures/`: W1 generated latency figures.
- `analysis/w2/tables/`: W2 grouped CSV/JSON summaries.
- `analysis/w2/figures/`: W2 generated energy figures.
- `analysis/w3/aggregates/`: W3 aggregate JSON outputs.
- `analysis/w3/tables/`: W3 generated CSV comparison tables.
- `analysis/w3/figures/`: W3 generated figures.
- `analysis/w3/reports/`: W3 generated Markdown reports.
- `analysis/merged/`: merged cross-run parquet datasets generated from raw run CSVs.

## Rule Of Thumb

If a file is generated by a script and can be recreated from `experiments/runs/` plus `docs/run_sheet.csv`, it belongs in `analysis/`. If a file explains how to run experiments or interprets results in prose, it belongs in `docs/`.
